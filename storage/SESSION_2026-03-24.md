## Code Changes

### `mpm_sim.py` — Rectangular grid + domain height support

Three new env vars, all backwards-compatible (defaults match previous hardcoded values):

| Env var | Default | Effect |
|---------|---------|--------|
| `JELLY_PARTICLES` | `80000` | Particle count per instance |
| `JELLY_GRID_Y` | `n_grid` (=128) | Grid cells in y-axis; rectangular grid if ≠ n_grid |
| `JELLY_DOMAIN_H` | `1.0` | Physical domain height |

Changes to simulation:
- `grid_v / grid_m` shape: `(n_instances, n_grid, n_grid_y)` — previously always square
- P2G and G2P boundary checks: `base[1] >= n_grid_y - 2` (was `n_grid`)
- Boundary damping: separate `damp_cells_x` (n_grid/20) and `damp_cells_y` (n_grid_y/20)
- Wall collisions: j uses `n_grid_y`
- Particle clamping: y uses `domain_height - 0.001` (was hardcoded 0.999)
- Render kernels (abyss, flat, vorticity): y mapped as `(domain_height - pos[1]) / domain_height`; x scaled by `/domain_height` to preserve aspect ratio
- Vorticity j-boundary: `n_grid_y - 1`

**Aspect ratio rendering:** With domain_height=2, x is compressed to 512 columns of the 1024×1024 frame buffer. The helper crops to `[:, :512]` to produce a correct 512×1024 portrait video.

The user's linter/editor also added per-instance timing fields in the same session:
- `instance_act_contraction` / `instance_act_refractory` — Taichi fields, per-instance timing genes
- `substep()` and `render_frame_abyss()` updated to use per-instance timing instead of global `ACT_CONTRACTION_END` / `ACT_RELAXATION_END`

### `make_jelly.py` — domain_height param

`fill_tank()` gained `domain_height=1.0` parameter. Water grid `wy` now spans `[margin, domain_height - margin]` instead of hardcoded `1.0`. Return signature changed to `(positions, materials, fibers, dict)` where dict contains `{n_robot, n_water, n_total, n_dead, muscle_count, self_intersecting}`.

`random_genome()` updated to 11 genes; genes 9/10 initialised at defaults (0.20, 0.40).

### `evolve.py` — tall flag, λ flexibility

- `--tall` pre-parsed flag sets `JELLY_GRID_Y=256`, `JELLY_PARTICLES=160000`, `JELLY_DOMAIN_H=2.0` before Taichi initialises
- `POPSIZE = sim.n_instances` — controlled entirely by `JELLY_INSTANCES`
- Ceiling cap: `domain_height - 0.07` (adapts to both 0.93 for square, 1.93 for tall)
- Spawn with `--tall`: `[0.5, 0.60]` (clears damping zone, more room above)
- Applies timing genes if `len(genome) > 9`

### `helpers/tall_tank.py` — new helper

Full render pipeline for the 128×256 tall tank:
- Sets env vars before import
- `fill_tank(..., domain_height=2.0)`
- Applies timing genes to `instance_act_contraction/refractory`
- Renders 512×1024 video (crops left half of 1024×1024 frame buffer)
- `--aurelia`, `--gen N`, `--genome [...]`, `--cycles N`, `--palette abyss/web`

**Test results:**

  - This genome would have saturated the ceiling at ~y=0.88 in the square tank
  - In the tall tank it swam freely past that point, confirming the tall tank removes the ceiling exploit

## Observations

- **Ceiling exploit confirmed experimentally:** best Exp 1 genome (`[0.248,...]`, fitness 0.535) swam to y=0.985 in 3 cycles in the tall tank — 0.095 units past where the square tank ceiling would have cut it off. The square-tank ceiling was directly shaping the evolved morphology.
- **Rendering aspect ratio:** the `pos[0] / domain_height` normalisation in render kernels is backwards-compatible — for domain_height=1.0 it is identical to the old `pos[0]`. Only the tall tank reveals the scaling effect.
- **Per-gen timing** for Exp 2 estimated at ~480s on a 3080 (4× Exp 1). The GPU memory footprint is ~370MB for 32×160K particles — well within 10GB VRAM.
